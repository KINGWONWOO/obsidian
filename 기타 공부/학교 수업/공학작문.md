# 1주차
---
### 공학 글쓰기 : 사직순특, 창상자일
- 기술이나 과학 분야에서 정보를 정확하게 전달하기 위한 글쓰기
- 기술 문서의 가장 큰 특징은 "특정"한 누군가를 대상으로 한다는 점
- 사직순특, 창상자일
![[Pasted image 20241014211308.png]]
### 기술 문서의 작성 단계 : 계구초검배
- 계획 세우기(10%) : 독자,목적
- 구조 잡기(10%) : 정보 배열
- 초안 작성(20%) : 모든 정보가 포함되게 문법이나 맞춤법 신경 X, 자신 있는 부분이나 중요한 곳부터 작성
- 검토와 수정(40%) : 내용-문서구조,문장연결,문법(내구연법) 등 모든 항목을 검토, 최종 테스트를 거쳐 배포본을 작성
- 배포(20%) : 독자에게 전달하는 방법 고려
### 기술 문서의 중요성
- 독자가 명확하지 않으면, 잘못된 정보 전달 + 이해하기 어렵 + 불쾌한 경험 전달
- 배포 방법도 독자에 따라 달라지므로 계획 단계에서 면밀하게 독자와 목적 정의

### 부정 행위 분류
- 표절 : 다른 사람의 것을 자신이 한 것처럼 작성
- 위조 : 없는 것을 있는 것처럼 
- 변조 : 결과를 고의로 누락하거나 변경

### 부정 행위 피하는 법
- 표절 방지 : 명시적으로 언급하고 인용부호 붙임
- 위조 방지 : 아이디어를 구체화하는 수준까지 글을 작성
- 변조 방지 : 예상과 다른 결과도 포함시키고 분석


# 2주차
---
### 알아야할 독자의 입장
- 독자의 지식
	- 글을 읽으면서 독자는 무엇을 학습하는가 (글의 구성/순서에 대한 이해)
- 독자의 의욕
	- 처음부터 끝까지 읽어주겠지 (치명적인 오판)
	- 노력해서 완성한 글이라도 확 넘겨버릴 수 있음 (내용에 집중)
- 독자의 목적
	- 독자가 목적에 부합하는 부분을 쉽게 찾을 수 있게(색인과 목차)

### 글의 기본
- 글은 형식과 내용으로 구성
	- 형식 : 어떤 제목, 장의 구성, 어떤 단어 쓸 것인가 등
	- 내용 : 글이 전달하는 의미
- 글의 내용이 중요하다. 그렇기에 더더욱 형식을 바르게 갖추어야 함.

### 글의 구조
- 정확하고 읽기 쉬운 글을 쓰기 위해서는 구조가 중요.
	- 부품 하나하나 정확하고 읽기 쉽게
	- 부품 간의 관계를 알기 쉽게

### 단어 주의사항
- 글꼴, 기호, 글말/입말
