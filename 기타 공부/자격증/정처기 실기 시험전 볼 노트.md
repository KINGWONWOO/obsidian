- PERT : 비관치/중간치/낙관치 3점 추점

### OSI 7계층 특징
- 응용 : HTTP, 데이터
- 표현 : 데이터
- 세션 : 데이터
- 전송 : TCP/UDP, 세그먼트
- 네트워크 : IP/ICMP, 패킷
- 데이터 링크 : 프레임
- 물리 계층 : 비트

### 디자인 패턴
#### 생성 패턴 :
**생빌 프로 팩앱싱**
	- Builder : 조립
	- Prototype : 원형 생성
	- Factory Method : 상위 클래스에서 객체 생성 인터페이스,                        하위 클래스에서 인스턴스 생성
	- Abstract Factor : 서로 연관되거나 의존적인 객체들의 조                         합
	- Singleton : 전역 변수를 사용하지 않고 객체 하나 생성
#### 구조 패턴:
**구 브데 퍼플 프로 컴 어**
	- Bridge : 계층을 연결
	- Decorator : 필요한 기능을 추가해 나감
	- Facade : 단순한 인터페이스를 제공
	- Flyweight : 경량화
	- Proxy : 실제 객체에 대한 대리 객체
	- Composite : 트리 구조
	- Adapter : 중간에서 맞춰주는 역할을 하는 인터페이스
#### 행위 패턴 :
**행 미인이 템옵 스테 비커 스트 메체**
	- Mediator : 중재자
	- Interpreter : 해석
	- Iterator : 안에 들어있는 모든 항목에 접근
	- Template Method : 서브 클래스로 캡슐화
	- Observer : 의존성, 느슨하게 결합
	- State : 객체 상태를 캡슐화
	- Visitor : 처리 기능을 분리하여 별도의 클래스를 만들어
	- Commnad : 각 명령이 들어오면 그에 맞는 서브 클래스가 선택/실행
	- Stratege : 알고리즘 군을 정의
	- Memento : Undo 기능
	- Chain of Responsibility : 동적으로 연결되어 있는 경우에 따라 다르게 처리될 수 있도록 연결

### 형상 관리
#### 형상 관리란
- 소프트웨어 개발을 위한 전체 과정에서 발생하는 모든 항목의 변경 사항을 관리하기 위한 활동 
#### 형상 관리 절차(식통감기)
- 형상 *식*별
- 형상 *통*제
- 형상 *감*사
- 형상 *기*록
#### 형상 관리 도구

- SubRoutine : 메인 루틴에 의해 필요할 때마다 호출되는 루틴

# 5단원 : 인터페이스 구현
---
### 용어 정리
- EAI : 기업에서 운영되는 서로 다른 플랫폼 및 애플리케이션 간의 정보를 전달,연계, 통합이 가능하도록 해주는 솔루션
- ESB : 기업에서 운영되는 서로 다른 플랫폼 및 애플리케이션들 간을 하나의 시스템으로 관리 운영할 수 있도록 서비스 중심의 통합을 지향하는 아키텍처
- JSON : '키-값 쌍'으로 이루어진 데이터 오브젝트를 전달하기 위해 인간이 읽을 수 있는 텍스트를 사용하는 개방형 표준 포맷
- AJAX : 자바스크립트를 사용하여 비동기적으로 XML 데이터를 교환하고 조작하 위한 웹 기술
- REST (Representaional State Transfer) : 웹과 같은 분산 하이퍼미디어 환경에서 자원의 존재/상태 정보를 표준화된 HTTP 메서드로 주고받는 웹 아키텍처
- IPSec : IP 계층에서 인증 헤더(AH)와 암호화(ESP)를 이용하여 보안 서비스를 제공하는 터널링 프로토콜
- SSL/TLS : 4-7계층에 클라이언트와 서버 간의 상호 작용 시 데이터 무결성을 보장하는 보안 프로토콜
- S-HTTP : 클라이언트와 서버 간에 전송되는 모든 메시지를 각각 암호화하여 전송

### DOS 공격 종류
- SYN Flooding : Hand-shake
- UDP Flooding : 대량의 UDP 패킷
- 스머핑(Smurfing) : 출발지 주소를 공격 대상의 IP로 설정하여 직접 브로드캐스팅 시킴
- 죽음의 핑(Ping Of Death) : 허용 범위 이상의 ICMP 패킷
- 랜드 어택 : 출발지IP = 목적지 IP
- 티어 드롭 : Offset
- BONK : 오류 제어

### 서버 접근 통제 유형
- 임의적 접근 통제(DAC) : 주체나 그룹의 신분에 근거
- 강제적 접근 통제(MAC) : 허용등급과 접근 허가 권환에 근거
- 역할 기반 접근 통제(RBAC) : 중앙 관리자가 사용자의 조직 내 맡은 역할에 기초

### 벨-라파듈라 모델(BLP)
- No Read Up, No Write Down

### 대칭 키 암호화 알고리즘
- DES(Data Encryption Standard) : 1975년 IBM 개발, NIST 발표
- SEED : 1999년 KISA
- AES : 2001년 미국 NIST
- ARIA : 2004년 국가정보원
- IDEA : DES 대체, 스위스 연방기술
- LFSR : XOR

### 비대칭 키 암호화 알고리즘
- 디피 헬만 : 최초
- RSA : 1977년 3명의 MIT
- ECC : 타원 곡선 암호
- ElGamal : 이산대수

### 해시 알고리즘
- MD5 : 1991년 Rivest MD4 개선
- SHA-1 : 1993년 미 정부 표준
- SHA-256 : SHA 알고리즘 한 종류
- HAS-160 : KCDSA(국내용)
- HAVAL : 메시지를 1024비트 블록 나눔

### 보안 위협
- XSS(Cross Site Script) : 검증되지 않은 외부 입력 데이터가 포함된 웹페이지
- 사이트간 요청 위조(CSRF) : 사용자가 자신의 의지와는 무관하게 공격자가 의도한 행위를 요청
- SQL 삽입(Injection) : 악의적인 SQL 구문을 삽입
- 허니팟 : 일부러 허술하게 만든 유인 시스템

### 화이트 박스 테스트

#### 종류
- 구문 커버리지 : 모든 명령문 적어도 한 번

- 조건 커버리지 : 개별 조건식이 적어도 한 번은 참과 거짓 결과 수행

- 기본 경로 커버리지 : 수행 가능한 모든 경로

#### 악성코드 관련 용어
- Watering hole : 자주 방문하는 웹 사이트를 사전에 감염
- C&C 서버 : 해커가 좀비 PC에 명령을 내리고 제어하는 서버
- 봇넷 : 감염된 다수의 컴퓨터들이 네트워크로 연결된 형태
- Sniffing : 네트워크에서 전송되는 데이터를 가로챔
- Pharming : 사용자가 의도하지 않은 웹 사이트로 리디렉션되는 공격
- SQL Injection : SQL 쿼리를 주입하는 공격
- Spyware : 개인 정보를 수집하거나 시스템을 모니터링
- Adware : 광고를 표시
- Evil Twin : 가짜 무선 액세스 WIFI 포인트 사용
- BlueBug : 블루투스 이용한 원격 제어 공격
- BlueSnarf : 데이터 도난
- BluePrinting : 이동 경로 추적
- BlueJacking : 익명의 메시지 전송
- Bluekeep : 원격으로 실행 가능한 취약점 이용
