# Git 사용법
## 1강 : git add, commit

| 명령어                     | 동작                          | 추가 설명                                          |
| ----------------------- | --------------------------- | ---------------------------------------------- |
| git init                | git 시작                      | + 최근 대부분의 프로그램에는 git이 내장되어 있으므로 잘 찾아보기         |
| git add 파일명             | git 파일을 Staging Area로 이동    | + 파일명은 여러 개 입력 가능<br>+ 파일명 대신 . 입력 시 모든 파일 add |
| git commit -m "메모"      | staging area에 있는 파일 저장소로 이동 |                                                |
| git status              | 현재 staging area에 있는 파일 확인   |                                                |
| git log --all --oneline | 현재까지 commit log 확인          | + --graph 추가해 branch 구별 가능                     |

## 3강 : git의 branch 만들기

| 명령어                | 동작                       | 추가 설명                   |
| ------------------ | ------------------------ | ----------------------- |
| git branch 브랜치명    | 새로운 branch 생성            |                         |

## 5강 : git revert, reset, restore

| 명령어                           | 동작                       | 추가 설명                                             |
| ----------------------------- | ------------------------ | ------------------------------------------------- |
| git restore 파일명               | 해당 파일은 최근 commit 상태로 복구  |                                                   |
| git restore --source 커밋ID 파일명 | 특정 commit 시점으로 파일 복구     |                                                   |
| git restore --staged 파일명      | 해당 파일 staging 취소(add 취소) |                                                   |
| git revert 커밋ID               | commit 취소                | + commit 삭제가 아닌. 해당 commit의 동작을 없애는 새로운 commit 생성 |
| git revert 커밋ID_1 커밋ID_2 ...  | commit 여러개 취소            |                                                   |
| git revert HEAD               | 최근 commit 취소             |                                                   |
| git reset --hard 커밋ID         | 특정 시점으로 리셋               | + 협업 시에는 조심히 사용                                   |
| git reset --soft 커밋ID         | 리셋인데 변동사항 지우지 말고 스테이징    |                                                   |
| git reset --mixed 커밋ID        | 리셋인데 변동사항 지우지 말고 unstage |                                                   |

# GitHub 사용법
## 1강 : git push
- 용어 정리
	- Git : 버전 관리 시스템
	- repository : Git이 파일 기록해두는 장소
	- GitHub : 온라인 원격 repository

| 명령어                        | 동작              | 추가 설명                                     |
| -------------------------- | --------------- | ----------------------------------------- |
| git push -u 원격저장소주소 로컬브랜치명 | 로컬 -> 원격저장소 업로드 | + -u 사용 시 뒤의 값을 기억해서 다음부턴 git push만 입력 가능 |
| git remote add 변수명 값       | 변수 문법 사용        | + 주소 저장에 사용 시 편함                          |

---
## 2강 : git clone, pull

| 명령어                   | 동작                   | 추가 설명                |
| --------------------- | -------------------- | -------------------- |
| git clone 원격저장소주소     | 원격저장소를 내 로컬에 불러오기    |                      |
| git pull 원격저장소주소 브랜치명 | 원격저장소 변경 사항 내 로컬에 적용 | pull = fetch + merge |
- 원격 저장소 상태와 내 로컬 저장소 상태가 다를 시 git push 불가.
- 이때는, pull한 후 push 해야함.(conflic 가능)

---
## 3강 : pull request
- pull/push만으로 협업이 가능하나, 개발자가 많아질 시에는 branch 생성 후 merge하는 것이 안전
- 기존 local 환경에서 branch 생성 후 commit.
- local 환경에서 merge 후 commit할 수 있으나. github 사이트의 pull request로 다른 사람들의 검토를 받아보는 것이 안전.
