## 4. Actor #액터
___
#### 4.1 액터의 구성
![[Pasted image 20240624190904.png]]
###### Actor는 Mesh(외형), Material(표면), Collision(물리 공간) 등의 다수의 Component로 구성

#### 4.2 액터 상태
- Static : 움직이지 않고 고정된 물체
- Stationary : 움직이지는 않지만 밝기나 그림자 방향 등은 변할 수 있는 물체
- Movable : 플레이 중에 움직일 수 있는 물체

#### 5.2 Material의 설정값들
![[Pasted image 20240624191151.png]]
- Base Color : 물체가 가진 고유한 색을 결정합니다.
- Metalic : 금속성
- Specular : 빛을 반사하는 정도. 0이면 반사없음 1이면 완전 반사
- Roughness : 표면의 매끄러운 정도
- Anisotropy : 미세한 결이나 홈
- Emissive Color : 자체 발광 효과
- Normal : 표면 효과
- Embient occlusion : 틈새 그림자 표현

#### 5.3 블루프린트 사용
- Material 설정 detail 패널에서 Shading Model의 설정 값을 바꿔서 빛을 계산하는지 여부를 설정 가능
- Material 설정에서 변수값 필요시 Scalar Parameter 노드 사용 가능
- Material 설정에서 Panner 노드 사용 시 위 아래/오왼 이어 붙이기 가능
## 6. Lighting #라이팅
___
- Lighting의 경우 Detail 패널의 Cast Shadow를 통해 그림자 설정 가능.
- LightMass를 통해 필요한 빛 계산을 미리 함. 이를 라이팅을 굽는다 표현.
- Directional Light는 태양과 같은 존재, Sky Light는 주변 환경을 캡처해서 환경광으로 사용하는 빛 유형-> 반사 재질 오브젝트에 주변 환경이 비춰질 때 사용
## 7. Collision #콜리즌
---

- 두 액터간의 방향을 가져올 때 벡터(Get Actor Location) 끼리의 빼기를 활용.
- Collision의 사용은 채널을 만든 후 해당 채널의 상호작용을 세팅하는 것으로 사용.
  이때 사용하는 설정은 Gnerate overlap Events, Collision Presets, Object Type 3가지
- Collision의 의도치 않은 동작 발생 시 액터의 모델이 되는 mesh에 collision 설정이 따로 있는지 확인.
